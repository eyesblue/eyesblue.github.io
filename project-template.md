# 專案模板

新增專案頁面時，可以直接沿用這個結構。

## 專案定位

簡短說明這個專案的用途、目標使用者，以及你為什麼要記錄它。

## 建置環境

- 作業系統
- 語言與版本
- 套件管理工具
- 必要帳號、金鑰或外部服務

## 建置流程

1. 取得原始碼
2. 安裝依賴
3. 設定環境變數或設定檔
4. 執行建置
5. 驗證輸出結果

## 部署流程

1. 準備部署環境
2. 上傳或發布
3. 驗證服務是否正常

## 常見問題

- 問題現象
- 原因
- 解法

## 參考資料

- 官方文件
- 相關 issue
- 自己的補充筆記
