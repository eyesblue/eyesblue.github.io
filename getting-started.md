# 寫作方式

這個網站適合把每個專案拆成獨立頁面，並維持一致的記錄格式。

## 建議每個專案都記錄的內容

1. 專案目的與使用情境
2. 開發或執行環境需求
3. 建置步驟
4. 部署步驟
5. 常見問題與排查方式

## 建議的頁面寫法

- 一個專案一頁，避免把不同案子的流程混在一起
- 共通知識另開分類頁，例如 Docker、Android、GitHub Pages
- 標題直接寫明工具、平台與版本，之後回頭查會比較快

## 新增專案時的做法

1. 複製「專案模板」頁面的結構
2. 改成新專案名稱
3. 填入實際環境、步驟與踩坑紀錄
4. 在「專案清單」補上連結
